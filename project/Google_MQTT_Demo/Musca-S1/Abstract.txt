Google MQTT Demo for Musca-S1 Development Board with WiFi Module

This demo application connects to Google IoT through MQTT,
and publishes messages.

It requires a registered device in the Google IoT (Google Cloud IoT Console).

1. Configure Google IoT connection settings in demo.c:
 - PROJECT_ID:   Project ID
 - CLOUD_REGION: Cloud region
 - REGISTRY_ID:  Registry ID
 - DEVICE_ID:    Device ID

2. Add device private key in pkey.h

3. Configure WiFi connection settings in socket_startup.c:
 - SSID:          network identifier
 - PASSWORD:      network password
 - SECURITY_TYPE: network security

4. Select target:
 - Musca-S1 ESP8266:  ESP8266 WiFi Module
 - Musca-S1 WizFi360: WizFi360 WiFi Module

5. Build the application

6. Connect the on board DAP-Link Debugger via USB

7. Open a Terminal on PC and select the COM port of the DAP-Link Debugger
   Configure baudrate = 115200

8. Run the application and observe the messages in the Terminal
