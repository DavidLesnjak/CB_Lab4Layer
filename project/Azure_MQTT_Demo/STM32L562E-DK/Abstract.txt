Azure MQTT Demo for STM32L562E-DK Board with WiFi Module

This demo application connects to Azure IoT Hub through MQTT,
sends messages and checks for receive confirmation.

It requires a registered device in the IoT hub (Azure portal).

1. Configure IoT Hub connection string in iothub_ll_telemetry_sample_mdk.c:
 - connectionString: Connection string - primary key

2. Configure WiFi connection settings in socket_startup.c:
 - SSID:          network identifier
 - PASSWORD:      network password
 - SECURITY_TYPE: network security

3. Select target:
 - STM32L562E-DK ESP8266:  ESP8266 WiFi Module (ARDUINO - USART)
 - STM32L562E-DK WizFi360: WizFi360 WiFi Module (ARDUINO - USART)
 - STM32L562E-DK ISM43362: ISM43362 WiFi Module (ARDUINO - SPI)

4. Build the application

5. Use default jumpers settings except set JP5 off (ARDUINO - SPI)

6. Connect the on board ST-Link Debugger via USB

7. Run the application and observe the messages in the Debug (printf) window
